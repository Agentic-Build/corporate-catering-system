# Implementation plans (historical)

These are point-in-time implementation plans and design docs captured as each
phase / feature was built. They are kept for context and are linked from the
CHANGELOG, but they are **not** the authority on current behaviour — some
reference tooling that has since been removed (e.g. `make dev-up` / kustomize
overlays, superseded by the Helm chart in `chart/tbite-platform`).

For current state, see:

- `README.md` — setup, layout, make targets
- `docs/architecture/` — ADRs + architecture specifications (kept in sync)
- `CHANGELOG.md` — what shipped, by theme
- the git log / GitHub PRs — authoritative fine-grained history
